## Why can't position 3 attend to position 5?

The previous lessons gave us scaled dot-product attention: query and key vectors produce scores, scores get scaled by $\sqrt{d_k}$, softmax turns them into weights, weights mix the values. There is one structural rule still missing. For next-token prediction, the model is given tokens $0, 1, \ldots, i$ and must predict token $i+1$. It is not allowed to peek at the answer. So when position $i$ computes its attention over the sequence, the model must mask out everything at index $> i$.

The mechanism is a lower-triangular mask applied to the score matrix *before* softmax: set $\text{scores}_{i,j} = -\infty$ for $j > i$. After softmax, those weights are zero. The post-softmax weight matrix is the strict lower triangle (with the diagonal) — every position attends only to itself and to the past.

## The formula

$$
\mathbf{M}_{ij} = \begin{cases} 0 & \text{if } j \le i \\ -\infty & \text{if } j > i \end{cases}, \qquad
\mathbf{W} = \operatorname{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}} + \mathbf{M}\right)
$$

The mask is added to the (already-scaled) score matrix. $-\infty$ propagates through softmax as $\exp(-\infty) = 0$.

def causal_mask(n):
    M = np.full((n, n), -np.inf)
    return np.tril(M)  # lower triangle, including diagonal, is 0; rest is -inf

scores = Q @ K.T / np.sqrt(d_k) + causal_mask(n)
weights = np.exp(scores) / np.exp(scores).sum(axis=-1, keepdims=True)
# weight[:, j > i] is now exactly zero`}

## Reading the result

Two things to internalize.

**The mask is a hard wall, not a soft suggestion.** Setting the score to $-\infty$ is binary: the post-softmax weight is *exactly* zero, not a small number. A soft penalty like $-10$ would also work (the post-softmax weight would be small but nonzero, leaking future information through the gradient). Causal masking is the difference between training a language model and training a language model that has memorized the answer.

**The lower triangle is the only structure the model can use.** For position $i$, the available keys are $0, 1, \ldots, i$ — exactly $i+1$ of them. The model has to build its next-token prediction from those. If the attention pattern is well-trained, the row of weights for position $i$ concentrates mass on whichever past position is most relevant (the previous word, a syntactic head, a long-range referent).

The cost of causal masking: every row of the score matrix throws away half of its information (the upper triangle). But this is what makes the model *generative* — capable of producing one token at a time without seeing the future. Bidirectional models (BERT-style) drop the mask and use both halves; they can only encode, not generate.

## Recap

Three takeaways. **For next-token prediction, position $i$ must attend only to positions $\le i$** — predicting token $i+1$ from tokens $i+1, i+2, \ldots$ is cheating. **A lower-triangular mask of $-\infty$, added to the score matrix before softmax, makes the post-softmax weights exactly zero above the diagonal.** Softmax turns $-\infty$ into 0 because $\exp(-\infty) = 0$. **The cost is structural: every row throws away half the score matrix** — the upper triangle is wasted, but only this half is what makes the model *generative*, capable of producing one token at a time without seeing the future.
