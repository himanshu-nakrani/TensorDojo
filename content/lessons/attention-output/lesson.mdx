## Once you have the attention weights, what do you do with them?

The previous lesson ended with the attention weight matrix $W$ — the row-wise softmax of the score matrix, with each row summing to one. The $W$ matrix says *how much* of each value vector to use. The next step is the **attention output**: the weighted sum of the value vectors.

For each query position $i$, the output is

$$
\text{out}_i \;=\; \sum_{j=0}^{n-1} W_{ij} \, \mathbf{v}_j
$$

$\mathbf{v}_j$ is the value vector for position $j$ — the "what to write" — and $W_{ij}$ is the attention weight from query $i$ to key $j$ — the "how much of this one." The result $\text{out}_i$ is the new representation of token $i$, a weighted mix of the value vectors.

## Reading the result

Three things to internalize.

**The output of attention is a per-token mix of value vectors.** Each token's new representation is a weighted sum of the value vectors; the weights are the attention row for that token. This is the "what the model says about each token, given the attention pattern" — the architecture has done a soft retrieval from the values.

**The spectrum of $W$ runs from one-hot to uniform.** One-hot $W$ means every output is a single value vector (the model is "looking at" exactly one position). Uniform $W$ means every output is the average of all values (the model is "looking at" everywhere equally). Real attention patterns live in between, and the temperature slider controls where on the spectrum the pattern lands.

**This is the full self-attention operation, end to end.** $Q \cdot K^\top$ → scaled → masked (if causal) → softmax → $W \cdot V$ → output. The output of attention is the input to the next sublayer (the FFN), and the same pattern repeats at the next block.

<Callout targetInteractive="attention-output-explorer">
  Switch back to "softmax" mode and drag the temperature slider from 0.1 to 3.0. The weight matrix sharpens at $T \to 0$ (every row of $W$ collapses to one-hot) and flattens at $T \to \infty$ (every row of $W$ approaches uniform). The output vectors smoothly trace the path between the two bookend behaviors. The temperature dial is the same dial from the attention-scores and softmax lessons — same parameter, same meaning, applied per row.
</Callout>

## A second look

The two bookend modes in the centerpiece — one-hot and uniform — are *not* realistic attention patterns, but they make the spectrum concrete. In a real trained model, the rows of $W$ are neither fully one-hot nor fully uniform; they have a few large entries and many small ones, and the temperature of softmax on the score matrix decides the contrast. The lesson's secondary exercise is exactly this: see the temperature move the weight matrix from "sharp" to "flat" and watch the output vectors smoothly trace the path between the bookend behaviors.

## Recap

Three takeaways. **The attention output is the weighted sum of value vectors** — $\text{out}_i = \sum_j W_{ij} \mathbf{v}_j$, with $W$ the row-stochastic weight matrix and $\mathbf{v}_j$ the value vector at position $j$. **V carries the content; W chooses the mix** — the value vectors are the model's "what to write," and the attention weights are the routing decisions. **The spectrum of $W$ runs from one-hot (every output is a single value) to uniform (every output is the centroid of all values)**, with realistic attention patterns in between and temperature controlling the contrast. The next lesson covers a numerical problem with the raw $Q \cdot K$ scores that the architecture has to fix before any of this works at non-trivial dimensions: the variance of the scores grows with $d_k$, which pushes softmax to saturate.
