## What if the same sentence has many simultaneous structures?

One attention computation is one similarity story. "The animal didn't cross because it was tired" — the word *it* refers to *the animal*, but it also has a syntactic relation to *was*. These are different questions about the same sentence, and one attention head can only answer one.

The trick: run several attention computations in parallel, each with its own learned Q, K, V projections. Concatenate the results. The model has *h* simultaneous "views" of the same sequence. The cost is roughly the same as one big attention; the expressive power multiplies because each head can specialize.

## The formula

Given $Q, K, V$ each of shape $[n, d_{\text{model}}]$, split $d_{\text{model}}$ into $h$ heads of size $d_k = d_{\text{model}} / h$:

$$
\text{head}_h = \operatorname{softmax}\!\left(\frac{Q_h K_h^\top}{\sqrt{d_k}} + M\right) V_h, \qquad
\operatorname{output} = \operatorname{concat}(\text{head}_1, \ldots, \text{head}_h) \cdot W_{\text{out}}
$$

Each head has its own learned $W_h^Q, W_h^K, W_h^V \in \mathbb{R}^{d_{\text{model}} \times d_k}$ that project the shared input into a head-specific subspace. The $W_{\text{out}}$ projection mixes the head outputs back into $d_{\text{model}}$.

## Reading the result

Three properties to internalize.

**Each head is a small attention with its own projection.** The total parameter count is roughly the same as a single-head attention with the same $d_{\text{model}}$, because the projections $W_h^Q, W_h^K, W_h^V$ are small ($d_{\text{model}} \times d_k$) and there are $h$ of them. The model pays for $h$ small projections instead of one big one — the FLOPs are similar, the capacity is $h$-way.

**Different heads learn different "questions" about the same input.** One head might attend to the previous token (a bigram model). Another might attend to syntactic heads. Another to long-range coreference. The model does not *tell* the heads what to do; training discovers useful decompositions. Empirically, the heads do specialize, and ablation studies show that some heads are far more important than others.

**The output projection mixes the head results back into $d_{\text{model}}$.** Without $W_{\text{out}}$, the next layer would see $h$ separate streams of dimension $d_k$ and have no way to use them as a single $d_{\text{model}}$-dimensional vector. $W_{\text{out}}$ is the layer's "read the heads and write a single thing" step.

A useful mental model: $h$ attention heads are $h$ small experts, each looking at the same input from a different angle, each producing a small write to a shared output. The model learns which expert to trust for which token.

## Recap

Multi-head attention is $h$ small attentions in parallel, each with its own learned Q, K, V projection. Cost stays roughly constant; the model gets $h$ independent views of the same sequence. In real implementations, the projections are concatenated into one big matrix to keep the implementation a single matmul.
