## What if the same sentence has many simultaneous structures?

One attention computation is one similarity story. "The animal didn't cross because it was tired" — the word *it* refers to *the animal*, but it also has a syntactic relation to *was*. These are different questions about the same sentence, and one attention head can only answer one.

The trick: run several attention computations in parallel, each with its own learned Q, K, V projections. Concatenate the results. The model has *h* simultaneous "views" of the same sequence. The cost is roughly the same as one big attention; the expressive power multiplies because each head can specialize.

## The formula

Given $Q, K, V$ each of shape $[n, d_{\text{model}}]$, split $d_{\text{model}}$ into $h$ heads of size $d_k = d_{\text{model}} / h$:

$$
\text{head}_i = \operatorname{softmax}\!\left(\frac{Q_i K_i^\top}{\sqrt{d_k}} + M\right) V_i, \qquad
\operatorname{output} = \operatorname{concat}(\text{head}_1, \ldots, \text{head}_h) \cdot W_{\text{out}}
$$

Each head $i \in \{1, \ldots, h\}$ has its own learned $W_i^Q, W_i^K, W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_k}$ that project the shared input into a head-specific subspace. The $W_{\text{out}}$ projection mixes the head outputs back into $d_{\text{model}}$.

## Reading the result

Three properties to internalize.

**Each head is a small attention with its own projection.** The total parameter count is roughly the same as a single-head attention with the same $d_{\text{model}}$, because the projections $W_h^Q, W_h^K, W_h^V$ are small ($d_{\text{model}} \times d_k$) and there are $h$ of them. The model pays for $h$ small projections instead of one big one — the FLOPs are similar, the capacity is $h$-way.

**Different heads learn different "questions" about the same input.** In a trained model, one head might attend to the previous token (a bigram view), another to syntactic heads, another to long-range coreference. The model is not told what each head should do; training discovers useful decompositions, and ablation studies show that some heads end up far more important than others. The interactive here cannot show that learning — the rotation sliders simulate what the parallel structure makes *possible*, not what training produces. Take the lesson as: the architecture leaves room for $h$ different stories, and trained models actually use that room.

**The output projection mixes the head results back into $d_{\text{model}}$.** Without $W_{\text{out}}$, the next layer would see $h$ separate streams of dimension $d_k$ and have no way to use them as a single $d_{\text{model}}$-dimensional vector. $W_{\text{out}}$ is the layer's "read the heads and write a single thing" step.

A useful mental model: $h$ attention heads are $h$ small experts, each looking at the same input from a different angle, each producing a small write to a shared output. The model learns which expert to trust for which token.

## Recap

Three takeaways. **Multi-head attention is $h$ small attentions in parallel, each with its own learned $W_i^Q, W_i^K, W_i^V$ projection.** Total parameter count is roughly constant — the projections are small and there are $h$ of them. **Different heads learn different "questions" about the same input**: in a trained model, one head might attend to the previous token, another to a syntactic head, another to a long-range coreferent. Training discovers useful decompositions; the architecture leaves the room. **The output projection $W_{\text{out}}$ mixes the head results back into $d_{\text{model}}$** — without it, the next layer would have no way to use the $h$ separate streams as a single vector. Stacking $h$-head blocks deep introduces a new problem the architecture has not yet solved — vanishing gradients — which the next lesson fixes with residuals and layer norm.
