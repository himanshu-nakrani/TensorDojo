## If attention is permutation-invariant, how does the model know the order?

The dot product $\mathbf{q} \cdot \mathbf{k}$ is a property of two vectors — it does not care which came first. A model that sees only embeddings would treat "the cat sat" and "sat cat the" identically. The order has to be smuggled in.

The transformer adds a position-dependent vector to each token's embedding *before* attention. The position vector has no learned parameters — it is a deterministic function of the index $0, 1, 2, \ldots$. The original paper used sin and cos at geometrically-scaled frequencies; that is the **sinusoidal positional encoding**.

## The formula

$$
\operatorname{PE}(\text{pos}, 2i) = \sin\!\left(\frac{\text{pos}}{10000^{2i / d}}\right), \qquad
\operatorname{PE}(\text{pos}, 2i + 1) = \cos\!\left(\frac{\text{pos}}{10000^{2i / d}}\right)
$$

The wavelength for dimension $i$ is $2\pi \cdot 10000^{2i / d}$. Low dimensions have short wavelengths (position 0 vs 1 vs 2 are distinguishable). High dimensions have long wavelengths (position 0 vs 1000 are distinguishable). The full table encodes every position uniquely.

def sinusoidal_pe(max_pos, d):
    out = np.zeros((max_pos, d))
    half = d // 2
    for pos in range(max_pos):
        for i in range(half):
            wavelength = 2 * np.pi * (10000 ** (i / half))
            out[pos, 2 * i] = np.sin(pos / wavelength)
            out[pos, 2 * i + 1] = np.cos(pos / wavelength)
    return out`}

## Reading the result

Two properties to internalize.

**Bounded.** Every entry of PE is in $[-1, 1]$. The encoding can never dominate the embedding; it nudges the embedding in a known direction with a known magnitude.

**Distance is a function of the dot product.** Two positions at indices $a$ and $b$ have a dot product that is a smooth, monotonic-ish function of $|a - b|$. Near positions have similar PE vectors and high dot products; far positions have very different PE vectors. This is what the model uses: when position $i$ attends to position $j$, the contribution $\mathbf{q}_i \cdot \mathbf{k}_j$ includes a term from the PE vectors that says "how far apart these are".

The sinusoidal scheme has a nice property that the original paper exploited: a shift of $k$ in position corresponds to a rotation in the PE space, and the dot product $\text{PE}(a) \cdot \text{PE}(a + k)$ is a closed-form function of $k$ and the dimension $i$. The model "knows" relative position without being told it explicitly.

## Recap

Three takeaways. **Positional encoding is a deterministic, parameter-free vector added to each token's embedding before attention.** The model has no way to learn "this is position 5" from the input alone, so the architecture injects it. **The sinusoidal scheme uses sin and cos at geometrically-scaled frequencies** — low dimensions encode nearby positions, high dimensions encode far-apart ones. **The pairwise dot products are a smooth function of position distance**, so the model can "feel" how far apart two tokens are without being told relative position explicitly. With position in hand, attention can finally answer the next question the architecture has been dodging: for next-token prediction, position $i$ is not allowed to see the future.
